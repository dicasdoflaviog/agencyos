OS 5 DOCUMENTOS MESTRES (Sempre sincronizados)

1. README.txt
   └─ Apresentação geral do projeto

   └─ Organograma de agentes (quem reporta pra quem)

CHECKLIST: ANTES DE CONSIDERAR IMPLEMENTAÇÃO "PRONTA"
═══════════════════════════════════════════════════════════════════════════════

Toda vez que você implementar algo novo:

□ Implementei / Testei / Funciona OK

□ Atualizei ARQUITETURA_COMPLETA.txt
  └─ Onde? (qual seção?)
  └─ O quê? (descrição clara)
  └─ Como integra? (com o resto do sistema)

□ Atualizei HIERARQUIA_AGENTES.txt (se afeta fluxo entre agentes)
  └─ Organograma reflete mudança?
  └─ Checklist de handoff atualizado?

□ Atualizei MANUAL_INTEGRACAO_SQUAD_V3.0.txt (se afeta fluxo de campanha)
  └─ Qual fase afeta? (01-05)
  └─ Qual agente afeta?

□ Atualizei GUIA_OPERACIONAL_BRIEFING.txt (se afeta briefing ou dados)
  └─ Coleta de dados mudou?
  └─ Fluxo cliente até live mudou?

□ Atualizei README.txt (se é relevante pra visão geral)
  └─ Descrição geral precisa mudar?

□ Todos os 5 documentos estão sincronizados?
  └─ Não tem contradição?
  └─ Fluxo está claro?

□ PRONTO PARA COMUNICAR ao time

FREQUÊNCIA DE REVISÃO DOS DOCUMENTOS
═══════════════════════════════════════════════════════════════════════════════

SEMANAL:
└─ Você implementou algo novo?
   └─ SIM → Atualizar os documentos que afeta
   └─ NÃO → Nada a fazer

MENSAL:
└─ Revisar MANUAL_INTEGRACAO_SQUAD_V3.0.txt
   └─ Está refletindo realidade de como squad trabalha?
   └─ Alguma mudança que acumulou sem documentar?

TRIMESTRAL:
└─ Revisar ARQUITETURA_COMPLETA.txt (visão geral)
   └─ Tudo ainda faz sentido?
   └─ Algum agente/framework que deveria ser removido?
   └─ Alguma integração que virou inviável?

SEMESTRAL:
└─ Revisão completa de todos os 5 documentos
   └─ Por Head + Estrategista
   └─ Tudo sincronizado?
   └─ Sistema está escalável?

REGRA DE OURO:

🎯 Se não tá documentado = não tá implementado

├─ O time não sabe

├─ Sistema não escala
└─ Verdade verdadeira: não funciona
