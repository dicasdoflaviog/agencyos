┌─────────────────────────────────────────────────────────────┐
│ COMPETÊNCIA 1: USABILIDADE & UX MOBILE-FIRST              │
├─────────────────────────────────────────────────────────────┤
│                                                              │

│ • Espaçamento: 16px entre elementos (legibilidade)        │
│ • Hierarquia clara: H1 > H2 > Body (sem confusão)        │
│ • Reduzir carga cognitiva: máx 3 opções por seção        │
│ • Touch-friendly: sem hover-only content                  │
│                                                              │
│ CHECKLIST:                                                  │
│ [ ] Layout funciona em 320px (iPhone SE)                  │
│ [ ] Todos botões clicáveis sem zoom                       │
│ [ ] Texto legível sem ampliar (tamanho mínimo 16px)      │
│ [ ] Navegação acessível com uma mão                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ COMPETÊNCIA 2: OTIMIZAÇÃO DE ATIVOS (Performance)        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│ FORMATOS OBRIGATÓRIOS:                                    │
│ • WebP: Imagens no padrão moderno (50% menor que JPG)    │
│ • SVG: Ícones e gráficos vetoriais (escaláveis)          │
│ • Fallback: JPG/PNG apenas para browsers antigos          │
│                                                              │
│ TÉCNICAS:                                                   │
│ • Lazy loading: Imagens carregam ao scroll                │
│ • Progressive JPEG: Imagem melhora conforme carrega       │
│ • WebP com fallback: <picture> tag obrigatória           │
│ • Tamanho de arquivo: máx 100KB por imagem               │
│                                                              │
│ CÓDIGO EXEMPLO:                                            │
│ <picture>                                                  │
│   <source srcset="hero.webp" type="image/webp">         │
│   <img src="hero.jpg" alt="[descrição]" loading="lazy">  │
│ </picture>                                                  │
│                                                              │
│ CHECKLIST:                                                  │
│ [ ] Todas imagens em WebP com fallback JPG              │
│ [ ] Ícones são SVG (não imagem)                          │
│ [ ] Lazy loading implementado                            │
│ [ ] Tamanho máximo de imagem <= 100KB                   │
│ [ ] LCP (Largest Contentful Paint) < 2.5s               │
│                                                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ COMPETÊNCIA 3: MICROANIMAÇÕES (Framer Motion/Tailwind)   │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│ PROPÓSITO: Dar vida à interface sem sacrificar performance │
│                                                              │
│ ANIMAÇÕES PERMITIDAS:                                     │
│ • Fade in/out (ao scroll, ao carregar)                   │
│ • Slide (elementos entrando de lado)                      │
│ • Scale (hover em botões/cards)                          │
│ • Stagger (items em sequência)                           │
│ • Pulse (ênfase em CTAs importantes)                     │
│                                                              │
│ ANIMAÇÕES PROIBIDAS:                                      │
│ ❌ Parallax pesado (mata performance em mobile)          │
│ ❌ Transições > 500ms (sente muito lenta)               │
│ ❌ Auto-play de vídeo (mata conversão)                   │
│ ❌ Animações infinitas distrayindo (neurotoxinas)        │
│                                                              │
│ PADRÃO RECOMENDADO:                                       │
│ • Duration: 300-500ms (sensação de rapidez)             │
│ • Easing: ease-in-out (natural)                         │
│ • Trigger: On scroll, on hover (user action)            │
│                                                              │
│ CÓDIGO (Tailwind Animate):                               │
│ <div className="animate-fadeIn duration-300 ease-in">   │
│   Conteúdo aparece suavemente                           │
│ </div>                                                     │
│                                                              │
│ CHECKLIST:                                                  │
│ [ ] Animações usadas apenas para ênfase                 │
│ [ ] Duração máxima 500ms                                │
│ [ ] Testado em device lento (throttling 3G)            │
│ [ ] Respeita prefers-reduced-motion                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘

// components/Hero.tsx
import Image from 'next/image';

interface HeroProps {
  title: string;
  subtitle: string;
  imageSrc: string;
  ctaText: string;
  ctaUrl: string;
}

export function Hero({
  title,
  subtitle,
  imageSrc,
  ctaText,
  ctaUrl,
}: HeroProps) {
  return (
    <section className="bg-white">
      <div className="max-w-6xl mx-auto px-4 py-16 grid grid-cols-1 md:grid-cols-2 gap-12 items-center">
        
        {/* Texto */}
        <div>
          <h1 className="text-5xl font-bold text-black mb-6 leading-tight">
            {title}
          </h1>
          <p className="text-xl text-gray-600 mb-8">
            {subtitle}
          </p>
          <a
            href={ctaUrl}
            className="inline-block bg-yellow-400 text-black px-8 py-4 font-bold rounded-lg hover:bg-yellow-500 transition-colors"
            aria-label={ctaText}
          >
            {ctaText}
          </a>
        </div>

        {/* Imagem */}
        <div className="relative w-full aspect-square">
          <picture>
            <source srcSet={imageSrc.replace('.jpg', '.webp')} type="image/webp" />
            <Image
              src={imageSrc}
              alt="Ilustração representando a solução de marketing"
              fill
              sizes="(max-width: 768px) 100vw, 50vw"
              priority
              className="object-cover rounded-lg"
            />
          </picture>
        </div>

      </div>
    </section>
  );
}

FUNÇÃO SOBRE ESTÉTICA
O design de interface não é para ser "bonito".
É a tradução visual da decisão estratégica do negócio.
Cada componente responde: "Isso facilita qual ação do usuário?"

ANTI-COMMODITY
Evitar padrões genéricos de "IA/template". A consistência técnica entre
componentes — tokens, spacing, hierarquia — é o que diferencia.

LANDING PAGES POR NÍVEL DO FUNIL
  Checklist (topo)      → Baixo atrito, CTA simples, sem campos desnecessários
  Manual (meio)         → Credibilidade, benefícios claros, prova social
  Diagnóstico (premium) → Premium, agenda/formulário qualificado, peso visual
  Estruturação (HT)     → Exclusivo, sem preço visível, CTA para conversa
